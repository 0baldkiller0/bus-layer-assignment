# 主要工作：面向 PCB 总线规划的层分配与可布通性反馈优化方法研究

时间充裕可以做：在反馈优化框架基础上，进一步提出结构化 Agent 决策模块，将布线失败反馈转化为可执行优化动作，实现多轮自适应优化。

> **主线：基于可布通性反馈的 PCB 总线层分配优化方法**
> **支线：面向 PCB 总线规划的工具增强型 EDA Agent**

前 6–7 个月先把第一选择做成一个完整可毕业课题；后面时间充足，再把 Agent 加进去，作为增强创新点，而不是一开始就把毕业押在 Agent 上。

---

# 一、总体路线

课题可以拆成两个版本。

## 最低限度版本

题目可以是：

> **面向 PCB 总线规划的层分配与可布通性反馈优化方法研究**

核心内容：

```text
输入 PCB 总线/网络/引脚/层信息
↓
构建总线冲突图与候选层分配
↓
初始层分配
↓
布线或可布通性评估
↓
根据失败反馈局部调整
↓
输出更优层分配结果
```

这个版本不依赖大模型，不依赖复杂 Agent，论文上更稳。

---

## 版本 B：增强版 / 时间充足后加入

题目可以升级成：

> **结合领域算法与智能体反馈的 PCB 总线层分配优化方法研究**

或者：

> **面向 PCB 总线规划的工具增强型 EDA Agent 研究**

增强内容：

```text
Agent 读取布线失败反馈
↓
判断失败类型
↓
选择调整动作
↓
调用层分配算法 / 局部重排算法 / 布线评估工具
↓
多轮优化
```

Agent 在这里不是主角，而是调度器。核心贡献仍然是 PCB 总线规划和层分配优化。

## 第 0–1 个月：明确问题边界与论文定义


### 目标


> 在 PCB 总线密集场景下，如何通过层分配和冲突消解，提高总线可布通性、减少层数、减少交叉和过孔。

### 主要任务

明确以下内容：

```text
1. 输入是什么？
   - 总线集合
   - 每条总线包含哪些 nets
   - 引脚位置
   - 板层数量
   - 障碍物或禁布区域
   - 设计规则约束

2. 输出是什么？
   - 每条 net / bus 分配在哪一层
   - 总线排序
   - 局部冲突调整结果
   - 可布通性评价结果

3. 优化目标是什么？
   - 最大化布通率
   - 最小化使用层数
   - 最小化总线交叉
   - 最小化过孔数
   - 最小化线长偏差
   - 控制运行时间

4. 和本科工作的区别是什么？
   - 本科：一次性总线合并 / 层分配 / 冲突处理
   - 硕士：加入可布通性反馈，根据失败结果迭代优化
```

### 阶段产出

到第 1 个月结束，应该有：

```text

metrics_definition.md
baseline_plan.md
论文初步目录
```

## 第 3–5 个月：完成主线创新——可布通性反馈优化

这是第一选择的核心阶段。

### 目标

在已有层分配算法基础上，加入 feedback loop。

本科的算法可能是：

```text
输入问题
↓
一次性层分配
↓
输出结果
```

要改成：

```text
输入问题
↓
初始层分配
↓
可布通性评估
↓
识别失败区域
↓
局部调整
↓
重新评估
↓
直到收敛或达到迭代上限
```

### 主要任务

需要定义几类“失败反馈”。

例如：

```text
失败类型 1：某一层冲突过多
失败类型 2：某两组 bus 交叉严重
失败类型 3：某个通道拥塞
失败类型 4：某组 bus 需要跨层次数过多
失败类型 5：局部区域障碍物导致不可布通
失败类型 6：某些 nets 被分配到不合理层
```

然后为每类失败设计调整动作：

| 失败类型       | 调整动作            |
| ---------- | --------------- |
| 层内冲突过多     | 将部分 bus 移到其他层   |
| 总线交叉严重     | 调整 bus ordering |
| 局部通道拥塞     | 重新分配该区域附近 bus   |
| 过孔过多       | 加入 via penalty  |
| 层数过高       | 合并低冲突 bus 到同一层  |
| 某 bus 多轮失败 | 提高该 bus 的分配优先级  |

### 算法结构建议

可以写成这样：

```text
Algorithm: Routability Feedback Layer Assignment

Input:
  PCB bus instance I
  maximum layer count L
  max iteration T

Step 1:
  Generate initial layer assignment A0

Step 2:
  Evaluate routability score R(A0)

Step 3:
  Detect conflict regions and failed bus groups

Step 4:
  Generate candidate local modifications

Step 5:
  Select best modification according to objective function

Step 6:
  Update assignment

Step 7:
  Repeat until convergence
```

### 目标函数建议

你可以设计一个综合 cost：

```text
Cost = α * conflict_count
     + β * crossing_count
     + γ * layer_usage
     + δ * via_count
     + ε * estimated_wirelength
     + ζ * unrouted_net_count
```

这样论文会更像一个正式优化问题，而不是工程 patch。

# 三、第 6 个月的关键决策点

第 6 个月是非常重要的分水岭。

你需要根据主线完成度决定后面是否进入 Agent 方向。

## 情况 A：主线结果已经明显优于 baseline

例如：

```text
布通率提升 10%–30%
冲突数下降明显
使用层数减少
过孔数降低
在复杂场景下优势明显
```

这时可以进入第二选择：

> 加入 Agent-guided iterative refinement。

## 情况 B：主线结果一般，但系统稳定

这时不要急着做 Agent。继续优化：

```text
改进失败反馈分类
改进局部调整策略
增加 ablation
扩大 benchmark
强化可视化
```

## 情况 C：主线还不稳定

这时必须停止扩展，专注保毕业：

```text
减少工具链复杂度
减少 Agent 设想
缩小问题规模
固定 benchmark
把算法和实验跑稳
```

我的建议是：**只有在第 6 个月主线已经能独立写论文时，才进入 Agent。**

# 四、第 6–8 个月：加入轻量 Agent 模块

这一阶段对应第二选择，但要做轻量版，不要做通用 EDA Agent。

### 目标

让 Agent 根据反馈报告选择下一步优化动作。

Agent 不直接改 PCB，不直接画线，只做策略选择。

### Agent 输入

Agent 可以看到结构化反馈：

```json
{
  "iteration": 3,
  "unrouted_nets": 12,
  "major_conflict_region": "region_2",
  "layer_conflicts": {
    "L1": 18,
    "L2": 7,
    "L3": 3,
    "L4": 11
  },
  "high_conflict_buses": ["BUS_A", "BUS_C"],
  "via_count": 52,
  "crossing_count": 31
}
```

### Agent 输出

Agent 只能输出合法动作：

```json
{
  "action": "local_layer_reassignment",
  "target_buses": ["BUS_A", "BUS_C"],
  "target_region": "region_2",
  "objective": "reduce_conflict"
}
```

你要限制它的 action space，不能让它自由发挥。

### 动作空间建议

定义 5–7 个动作就够了：

```text
Action 1: 局部层重分配
Action 2: 总线顺序调整
Action 3: 高冲突 bus 优先布线
Action 4: 降低某层拥塞
Action 5: 允许局部增加过孔
Action 6: 尝试减少使用层数
Action 7: 回滚到上一轮较优状态
```

### 对比实验

Agent 模块的对比不要太复杂，可以比较：

```text
1. 无反馈算法
2. 规则驱动反馈算法
3. 随机动作反馈算法
4. LLM Agent 选择动作
5. Ours: 结构化 Agent + 领域算法
```

重点不是证明 LLM 多聪明，而是证明：

> 结构化 Agent 能够在多轮优化中更快找到有效调整策略。

### 阶段产出

第 8 个月结束时，你应该有：

```text
Agent 状态表示
Agent 动作空间
JSON schema 校验
多轮优化日志
Agent 与非 Agent 的对比实验
```

```text
估计过孔
估计拥塞
输出可布通性评分
```

重要程度：最高。
完成时间：第 1–2.5 个月。

你的论文最后能不能站住，很大程度取决于 evaluator 是否定义清楚。

---

## 模块 4：Baseline 算法

负责内容：

```text
随机层分配
贪心层分配
图着色层分配
本科算法整理
```

重要程度：高。
完成时间：第 2–3 个月。

没有 baseline，就无法证明你的方法有效。

---

## 模块 5：可布通性反馈优化算法

负责内容：

```text
识别失败模式
生成局部修改候选
选择最优修改
迭代优化
保存每轮结果
```

重要程度：最高。
完成时间：第 3–5 个月。

这是第一选择的核心创新。

---

## 模块 6：工具链验证

负责内容：

```text
可选接入 Freerouting
可选接入 KiCad
真实或半真实 PCB case 验证
布线结果统计
```

重要程度：中等。
完成时间：第 5–6.5 个月。

这个模块不要喧宾夺主。它是验证工具，不是你的核心贡献。

---

## 模块 7：Agent 增强模块

负责内容：

```text
定义 Agent 状态
定义 Agent 动作空间
定义 JSON schema
让 Agent 根据反馈选择优化动作
与规则策略对比
```

重要程度：中等偏高。
完成时间：第 6–8 个月。

这是第二选择，也是论文亮点，但不能影响主线毕业。

---

## 模块 8：实验与论文

负责内容：

```text
总体对比实验
复杂度实验
消融实验
案例可视化
论文写作
答辩准备
```

重要程度：最高。
完成时间：贯穿全程，第 8 个月后集中完成。

# 十一、最小可毕业版本应该长什么样

到第 6–7 个月，你至少要保证有这个版本：

```text
1. 一个 PCB 总线层分配问题建模
2. 一个合成 benchmark
3. 三个以上 baseline
4. 一个可布通性反馈优化算法
5. 完整评价指标
6. 实验证明优于 baseline
7. 可视化案例
8. 论文主体可写
```

这个版本即使没有 Agent，也可以毕业。

论文贡献可以写成：

```text
1. 提出 PCB 总线层分配的可布通性反馈建模方法
2. 提出基于失败模式识别的局部重分配策略
3. 提出迭代式总线层分配优化算法
4. 构建面向 PCB 总线规划的实验 benchmark 和评价体系
```

这四点已经够支撑一个硕士课题。

---

# 十二、Agent 增强版应该长什么样

时间充足后，再加入这个版本：

```text
1. Agent 不直接布线
2. Agent 不直接修改 PCB 文件
3. Agent 只根据反馈选择动作
4. 所有动作经过规则校验
5. 真正执行仍由你的算法完成
```

Agent 的论文贡献可以写成：

```text
在反馈优化框架基础上，进一步提出结构化 Agent 决策模块，将布线失败反馈转化为可执行优化动作，实现多轮自适应优化。
```

注意，这个说法比“基于大模型的 EDA Agent 自动布线系统”稳很多。

# 十四、每周工作节奏建议

比较稳的节奏是：

```text
每周 3 天写代码
每周 1 天跑实验和整理结果
每周 1 天读论文 / 写文档
每两周向导师汇报一次
每月固定产出一份阶段报告
```

不要连续几个月只写代码不写文档。硕士课题很容易出现“代码做了很多，但论文不好写”的问题。

每个月至少整理一次：

```text
当前完成了什么
实验结果如何
遇到什么问题
下个月计划做什么
哪些内容可以写进论文
```

# 十五、风险控制策略

## 风险 1：Agent 做不出来

处理方式：

```text
Agent 只是增强模块
主线论文不依赖 Agent
第 6 个月后再决定是否加入
```

---

## 风险 2：真实 KiCad / Freerouting 接入太慢

处理方式：

```text
先用抽象 evaluator 完成主实验
工具链只做补充验证
不要让工具链成为主线瓶颈
```

---

## 风险 3：算法提升不明显

处理方式：

```text
增加复杂 benchmark
突出高密度 bus 场景
做分场景分析
强调在复杂约束下的改进
```

很多算法在简单场景下差别不大，真正优势往往体现在：

```text
bus 多
层数少
障碍物密集
引脚密度高
总线交叉严重
```

---

## 风险 4：论文创新点被质疑

处理方式：

不要只说“我用了反馈”。要把创新点说成：

```text
1. 面向 PCB 总线层分配的可布通性反馈建模
2. 基于失败模式识别的局部重分配策略
3. 多目标代价函数驱动的迭代优化算法
4. 可选：结构化 Agent 策略选择机制
```

这样听起来就是方法创新，而不是简单工程实现。

---

# 十六、最终推荐执行顺序

最稳的执行顺序是：

```text
第一步：做问题建模
第二步：做 benchmark 和 evaluator
第三步：复现本科算法
第四步：做反馈优化算法
第五步：做完整实验
第六步：写论文主线
第七步：时间充足再加 Agent
第八步：把 Agent 作为增强章节
```

不要一开始就做：

```text
LLM Agent
KiCad 插件
完整自动布线系统
OpenROAD
端到端 PCB 生成
```

这些都容易拖慢毕业。

---

# 十七、最推荐的最终题目表述

最稳的题目：

> **面向 PCB 总线规划的层分配与可布通性反馈优化方法研究**

时间充足后的升级版：

> **结合智能体策略选择的 PCB 总线层分配与可布通性反馈优化方法研究**

更偏 EDA Agent 的版本：

> **面向 PCB 总线规划的工具增强型 EDA Agent 研究**

我的建议是：**开题时用第一个题目，论文后期根据 Agent 完成度决定是否升级到第二个。**

这样你的课题既稳，又保留了往前沿方向扩展的空间。
